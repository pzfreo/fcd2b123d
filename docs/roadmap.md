# Roadmap

Prioritised PM view of open work. The ordering is product-impact-first:
*what improves the value of the translator to its actual users?* Effort is
a tie-breaker, not the primary axis.

> **Live progress**: see [`STATUS.md`](../STATUS.md) at the repo root
> for what's in flight, what's recently merged, and what's deferred.
> Autonomous-mode work logs non-obvious calls to
> [`docs/autonomous-decisions.md`](./autonomous-decisions.md).

Last updated: 2026-05-16 (post PR #79).

## Decision lens

The project's value proposition is "hand-readable parametric build123d
Python you can modify" — not "STEP round-trip". So issues are ranked
against:

1. **Output quality**: does it make the emit look more like hand-written
   bd_warehouse-grade build123d?
2. **Trust**: does it fix a "translates but produces wrong / brittle
   geometry" failure?
3. **Coverage**: does it unblock a meaningful fraction of the
   ~3,150-file library?

Other axes (developer ergonomics, debug tooling) are explicitly secondary
for v1.

---

## Top 3 — highest impact, do these first

### 🥇 #75 — Detect uniform patterns, emit `with PolarLocations`/`GridLocations`

**Why first**: this is the single biggest "doesn't look hand-written"
tell in our output (documented in `docs/design/emit-code-quality.md`).
Affects every pattern emit. Effort is ~1 day. Fixtures already exist
(`tier4_patterns/polar_pattern_holes`, `linear_pattern_holes`).

ROI = enormous perceptual quality jump per engineer-hour. Should be
unblocked immediately.

### 🥈 #76 — Use FreeCAD Labels for variable names, docstrings, and comments

**Why second**: second-biggest perceptual quality tell. Shifts output
from machine-named (`sketch_001`, `pocket_002`) to engineer-named
(`hex_head_profile`, `bolt_clearance_pocket`). 1-2 day effort. Touches
every fixture's emit.

After #75 + #76 land, a casual reader of a translated `.py` should not
be able to identify it as autogenerated at a glance.

### 🥉 #36 — Fillet/Chamfer face-adjacency edge selection

**Why third**: trust/coverage. The Oven_builtIn fixture represents a
*class* of failures (any deep dressup cascade), not a one-off bug. As
we add more tier-3 fixtures the midpoint-based edge selection gets
brittler. Also unblocks investigation work on #38 (post-Midplane bugs
are often fillet drift in disguise).

2-3 day refactor — biggest of the top 3 but the highest-leverage trust
fix. Decoupled from #75/#76 (different code paths), so can run in
parallel.

---

## Tier 2 — high value, defer until top 3 land

### #43 — Coherent snap for solver-noise coordinates

Cosmetic-but-pervasive. Every emit with sketch geometry has
`54.9999786…` digits the user typed as `55`. Naive snap breaks BRep
validity; needs anchor-point refactor in `sketch.py`. 1-2 days but
high-risk regression surface.

Tier 2 because it's cosmetic — the geometry is right, output is just
harder to read. Top 3 fixes the bigger perceptual issues first.

### #77 — Move runtime helpers to importable `fcstd2b123d.runtime`

Half-day cleanup. Quality-of-life for consumers translating
multi-file directories (no more 30 lines of inlined helpers per file).
Low risk, gated on opt-in CLI flag.

### #33 — Part::Helix

Half-day enhancement, 1.4% library coverage. Cheap, but only unblocks
1 corpus fixture. Layered on #34's Sweep support which already shipped.

## Tier 3 — defer / investigate

### #38 — Post-Midplane geometric mismatch investigation

3 excluded corpus fixtures (Foot + 2 unnamed). Likely interacts with
#36 (fillet drift) once that lands. Estimate 1-2 days per fixture.
**Underspecified**: the "2 others" need to be named explicitly before
this is actionable.

### #60 — WallHungBidet Hausdorff fail

Last remaining of the original three; cabin_door + FootPAD addressed
by the downsampling fix in PR #72. Real geometric difference (99 mm
vs 72 mm tolerance, bbox 721 mm). Needs feature-by-feature bisection.

### #78 — Opt-in builder-mode emit

2-3 day alt-style emit. Big visible jump in output quality if landed,
but #75+#76 already get us most of the way to bd_warehouse style
without changing the architectural contract. Re-evaluate priority
after #75/#76 ship — if those close the gap, #78 may slip to v2.

### #18 — Isometric render gallery

Debug tooling. Project initiative — needs renderer choice decision
(OCCT-native vs vtkplotter vs pyrender) before it's actionable. Not
on the critical path for v1 quality.

## Sequencing recommendation

```
Sprint 1 (≈1 week):       #75 → #76 → #77
Sprint 2 (≈1 week):       #36 → #43
Sprint 3 (investigation): #38 + #60 (parallel)
Tier 3 / v2:              #78, #33, #18
```

After Sprint 1 the emit quality should pass the "could a human
mistake this for hand-written?" test for tier-1 through tier-4
fixtures. After Sprint 2 we should have the trust story solid.

## Issues that need *better tests* (not just fixtures)

Most open issues have a *fixture* (a `.FCStd` that triggers the gap),
but several lack an *emit-source assertion* — a test that confirms the
emitted Python is *good*, not just that it produces the right geometry.
The corpus suite verifies geometry equivalence; the issues below need
something more.

### #75 Locations contexts — needs **emit-source assertion**

- Existing: `polar_pattern_holes` / `linear_pattern_holes` fixtures
  pass geometry-equivalence.
- **Missing**: assert that the emit for these fixtures contains
  `with PolarLocations(` / `with GridLocations(` and does *not*
  contain spelled-out `Rot(Z=60), Rot(Z=120), …` lists.
- Why: a sloppy implementation could emit *both* (chained Rots inside
  a Locations block) and pass the geometry check. The point of the
  issue is the *shape of the code*, which only a source-level test
  catches.

### #76 Labels for naming — needs **emit-source assertion**

- Existing: hex cap screw fixture verifies geometry.
- **Missing**: assert that the emit for a fixture whose FreeCAD doc
  uses Labels (e.g. hex cap screw) contains the Label strings as
  variable names, not `sketch_001` / `pocket_002`.
- Why: same as #75 — geometry is unaffected by naming; need a code-
  quality assertion.

### #43 Coherent snap — needs **emit-source assertion**

- Existing: `partdesign_example` fixture (geometry-only).
- **Missing**: assert that the emit for fixtures known to have
  solver near-misses (e.g. `partdesign_example.FCStd`) contains
  *no* float literals with >8 significant digits that don't trace
  to a user-typed irrational value. Or simpler: assert
  `format_value("54.9999786…")` rounds to `"55"` *after* the
  anchor-point refactor lands.

### #77 Shared helpers — needs **importable-emit test**

- **Missing**: with `--shared-helpers`, exec the emitted source and
  assert it imports from `fcstd2b123d.runtime` (and runs end-to-end
  against the installed runtime).
- The default (inlined) path is fully covered by existing tests.

### #78 Builder mode — needs **parametrised tier tests**

- **Missing**: parametrise the tier-1 through tier-4 tests over
  `--style=algebra` and `--style=builder`. Both must produce the
  same geometry.
- Bonus: a snapshot test on a representative fixture confirming the
  builder-mode emit contains `with BuildPart()` / `with BuildSketch()`
  and not `Sketch() + Plane(...) * (...)`.

### #38 Post-Midplane — needs **named fixtures**

- Issue body cites "Foot, plus 2 others where Midplane support landed
  but downstream features mismatch" — the 2 others are **not named**.
- **Required before action**: identify the 2 unnamed fixtures by stem,
  cite their paths, add them to `EXCLUDED_FROM_TEST` with the issue
  number. Without this the issue is a placeholder per the project
  CLAUDE.md fixture rule.

### #60 — partial; one fixture left

- WallHungBidet is the remaining failure. The fixture already exists
  in `sample_813` and is in `EXCLUDED_FROM_TEST`.
- **Missing**: a focused bisect test (truncated translation up to
  feature N) to identify which feature introduces the drift. The
  general corpus suite hits the symptom but not the bisected cause.

### #18 — out-of-band tool

- Not a regular test concern. Needs a smoke test (`render_gallery.py
  fixture.FCStd` produces non-empty PNG of expected size) once the
  renderer is chosen, but not corpus-suite coverage.

### #36, #33 — fixtures sufficient

- Both have clear fixtures (`Oven_builtIn` for #36, `Beam-coupling`
  for #33) currently in EXCLUDED. Standard "fixture flips from
  excluded to passing on close" applies. No additional test types
  needed.
